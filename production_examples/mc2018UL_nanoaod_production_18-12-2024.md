# NanoAOD production campaign for 2018 MC production

## Used configuration and setup:

* conditions: [`configuration/conditions.yaml`](../configuration/conditions.yaml)
* cmsdriver settings for nanoAOD: [`configuration/cmsdriver_nanoaod_specifics.yaml`](../configuration/cmsdriver_nanoaod_specifics.yaml)
* datasets: [`configuration/datasets_miniaod_nonResHH_background_2018UL.yaml`](../configuration/datasets_miniaod_nonResHH_background_2018UL.yaml)
* crab config template: [`crab_configuration/crab_template.py`](../crab_configuration/crab_template.py)
* CMSSW release: CMSSW_13_0_21

## Creation of CMSSW and Crab configs call

General call:

```bash
./create_configs.py --work-directory /ceph/$(whoami)/htt_mc_2018_crab_nanoaod_submission_18-12-2024_prodreleasev12/ \
  --datasets configuration/datasets_miniaod_nonResHH_background_2018UL.yaml \
  --conditions configuration/conditions.yaml \
  --cmsdriver configuration/cmsdriver_nanoaod_specifics.yaml \
  --nThreads 4 --numCores 4 \
  --maxMemoryMBperCore 1000 --publication --splitting EventAwareLumiBased --unitsPerJob 500000 --maxJobRuntimeMin 900
```

### Adaptations during production campaign

Some of the datasets were initially produced with an inconsistent number of events. To do a more reliable iteration, these input datasets
were moved to T1_DE_KIT_Disk, and the tasks were re-submited with an adapted config, restricting them to run at KIT via `--siteWhitelist T1_DE_KIT`.
This includes running the jobs at nearby sites `T2_DE_DESY` and `T2_DE_RWTH`.

Some jobs needed resubmission with longer maximum runtime.

The production was run in 2 parts, the latter of them supplementing the campaign with a few missing datasets.

## Managing of crab tasks call

Initial calls will be presented in the following, which can be adapted further for resubmission supplemented with available resubmission options (see `./create_configs.py --help`).

It is best to submit not all configs at onces (as indicated below), but in bunches, since the crab server seems to have an internal limit of 100 task to handle for a user in parallel.

```bash
./crab_manager.py --crab-config-pattern \
  /ceph/$(whoami)/htt_mc_2018_crab_nanoaod_submission_18-12-2024_prodreleasev12*/crabconfigs/*.py
```
